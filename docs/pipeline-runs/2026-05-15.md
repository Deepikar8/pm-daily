## Pass 0 — Thesis brief
- Central tension: "Descriptive, safe-feeling names (Microsoft Cloud Services, Infoseek, TripActions) vs. original, surprising names (Azure, Google, Navan). The descriptive option feels responsible to defend in a meeting but underperforms because it leaves the brand indistinguishable."
- Ideas (5):
  1. `never-describe` — "Anyone can describe what a company does — that name will have a thousand competitors that sound the same. Aim for original, surprising, invented." [central]
  2. `build-for-trust` — "First pillar; trust drives early momentum, which is now the single biggest factor for new-brand survival." [central]
  3. `sound-and-accessibility` — "Brain doesn't process complexity; familiar components + clean sound (Azure's 'z'; Subaru Outback) win." [secondary]
  4. `naming-as-strategic-not-clerical` — "Don't delegate naming to junior staff or run group-brainstorm votes; treat as strategic call." [central]
  5. `name-as-flexibility` — "Descriptive names cap your growth into new categories (TripActions → Navan; Microsoft Cloud Services → Azure)." [secondary]

## Pass 0 — Thesis brief
- Central tension: PMs hold scattered context in their own heads (Slack/Notion/decks/dashboards), and synthesis-in-head burns them out → vs → offload storage and synthesis to a ChatGPT Project that becomes a "second brain" — but keep judgment, taste, and the call with you.
- Ideas (6):
  1. `offload-context-not-judgment` — Second brain stores info & synthesizes; you still own judgment, taste, decisions. [central]
  2. `everything-is-text` — Decks/dashboards/Slack channels → PDF → upload. The "whoa" moment is realizing all PM input is text. [central]
  3. `build-the-personality-first` — Define instructions/personality before feeding files. Make it challenge you, not agree with you. [central]
  4. `feed-the-living-thing` — Continuously add new artifacts (your own PRDs, threads, screenshots) so the brain stays current. [secondary]
  5. `context-reload-cost` — Each audience reframe (marketing blurb / QBR / sales deck / UX script) used to cost a context reload. Second brain eliminates that. [central]
  6. `prototype-from-conversation` — Move from abstract discussion → Lovable prompt → working prototype in minutes; share with eng/design before code. [secondary]
